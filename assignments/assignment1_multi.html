Overview & dataset

This multimodal track studies food classification from paired image + text data. The dataset contains real food images together with textual descriptions, which satisfies the course requirement that multimodal data must be a true image–text pair rather than an artificial concatenation.

We use UPMC Food-101 with 101 food classes. The metadata is loaded from CSV files, then each sample is resolved to an image path, class name, textual description, and numeric label. This shared metadata layer is reused for EDA, zero-shot evaluation, few-shot sampling, and the final supervised multimodal head.

02

CLIP foundation & multimodal pipeline

The core model is OpenAI CLIP, which uses one encoder for images and one encoder for text, then projects both modalities into the same latent space. This shared representation allows us to compare foods using image evidence, text evidence, or a fusion of both.

In our module-first implementation, each sample starts from metadata in CSV form, is transformed into a tuple (img_path, label, text, class_name), then passes through a PyTorch Dataset and DataLoader. Images are converted to CLIP pixel values, text is tokenized by CLIPProcessor, and both are encoded to 512-dimensional embeddings before entering one of three downstream strategies.

Downstream strategies

Zero-shot No training
Prompt-based fusion
We compare image and text embeddings against class prompts and combine both similarities with a weighted score: α · sim(image) + (1 − α) · sim(text), using α = 0.5.
Main benefit Fast baseline
Few-shot Low-data
CLIP + Logistic Regression
We concatenate image and text embeddings into a 1024-D multimodal vector and train a linear LogisticRegression classifier with K ∈ {1, 5, 10} samples per class.
Main benefit Strong trade-off
Extension Best
Fine-tuned Multimodal Head
CLIP is kept frozen, while a compact MLP head is trained on cached image/text features. This gives a supervised multimodal classifier with strong performance and practical deployment cost.
Main benefit Highest accuracy
03

Comparative results

Following the course requirement for multimodal classification, we first compare zero-shot and few-shot approaches on the same CLIP backbone. We then report a stronger supervised extension, the fine-tuned multimodal head, as an additional comparison for performance analysis.

Method Accuracy Macro-F1 Interpretation
Zero-Shot CLIP (multimodal, α = 0.50) 0.8296 0.8337 Strong baseline without extra training
Few-Shot CLIP K = 1 + Logistic Regression 0.5731 0.5538 Too little supervision to separate classes well
Few-Shot CLIP K = 5 + Logistic Regression 0.8048 0.8095 Approaches zero-shot with limited labels
Few-Shot CLIP K = 10 + Logistic Regression 0.8517 0.8537 Best few-shot setting; surpasses zero-shot
Fine-tuned Multimodal Head 0.9383 0.9380 Best overall result

Main takeaway. Zero-shot CLIP already provides a strong multimodal baseline with no additional training. However, once a small number of labeled examples per class is available, few-shot learning improves rapidly. At K = 10, the few-shot classifier overtakes zero-shot. The supervised multimodal head remains the best model overall, reaching 93.83% accuracy and 93.80% macro-F1.

Error analysis

Most remaining errors come from visually and semantically similar dishes, which makes Food-101 a fine-grained classification task. The model often confuses food classes that share ingredients, plating style, or raw-food appearance.

  • Japanese dishes such as sushi and sashimi are repeatedly confused, which is consistent with their visual similarity in real food photos.
  • Several errors occur between appetizer-style dishes and plated cold dishes, suggesting that local texture cues alone are not always enough for perfect discrimination.
05

Extension: deployment-oriented design

Besides model comparison, the multimodal part is organized in a module-first way. The codebase separates data_loader, baselines, trainer, service, and web_api, so the notebook is used mainly for experimentation and visualisation while the underlying system remains ready for integration into a lightweight web demo.

This design matches the course suggestion of building a small demo application. In our setup, a user can upload a food image, provide a short text description, and receive predictions from zero-shot, few-shot, or the fine-tuned head through the same interface.

Conclusion. For the assignment requirement, zero-shot and few-shot already provide a valid and meaningful multimodal comparison. As an extension, the fine-tuned multimodal head offers the strongest final performance, while the module-first pipeline supports future deployment on GitHub Pages documentation plus a live demo backend.
